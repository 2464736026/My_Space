## 🔍 问题定位

根据持续的500错误，最可能的原因是：

### PDF库依赖问题（90%可能性）

`pdfplumber` 需要系统级依赖：
- libjpeg、libpng、zlib、freetype等

Vercel的Python运行时可能不包含这些库。

## 📋 你需要做的（按优先级）

### 🔴 优先级1：查看Vercel日志（最重要！）

**这是找到问题根源的唯一方法！**

1. 访问：https://vercel.com/dashboard
2. 进入项目：`my-space`
3. 点击 "Deployments" 标签
4. 点击最新的部署（时间戳最近的）
5. 查看两个标签：
   - **Build Logs**：构建日志
   - **Function Logs**：运行时日志（最重要！）
6. **复制所有日志内容**
7. **发送给我**

日志会告诉我们：
- 具体哪个模块导入失败
- 缺少什么依赖
- 完整的错误堆栈

### 🟡 优先级2：测试API端点

即使返回500，也尝试访问不同的端点：

```powershell
# 测试根路径
Invoke-WebRequest -Uri "https://my-space-beryl.vercel.app/" -UseBasicParsing

# 测试健康检查
Invoke-WebRequest -Uri "https://my-space-beryl.vercel.app/api/health" -UseBasicParsing

# 测试环境变量
Invoke-WebRequest -Uri "https://my-space-beryl.vercel.app/api/test" -UseBasicParsing
```

查看返回的错误信息是否有变化。

### 🟢 优先级3：确认环境变量

在Vercel Dashboard中：
1. 进入项目设置 (Settings)
2. 点击 "Environment Variables"
3. 确认这三个变量存在且值正确：
   - `OPENAI_API_KEY`
   - `OPENAI_MODEL`
   - `OPENAI_BASE_URL`
4. 确认它们应用到了 "Production" 环境
5. 截图发送给我（可以隐藏API key的值）

## 🎯 可能的解决方案

### 方案A：移除pdfplumber，只用PyPDF2

如果确认是PDF库问题，我可以：
1. 移除pdfplumber依赖
2. 只使用PyPDF2（纯Python实现，无系统依赖）
3. 可能解析质量稍差，但能在Vercel上运行

### 方案B：切换部署平台

如果Vercel确实不支持，可以切换到：
- **Railway**：支持更完整的依赖，部署简单
- **Render**：免费层支持Python，依赖支持好
- **Fly.io**：支持Docker，完全控制环境

### 方案C：前端提取PDF文本

使用JavaScript库（pdf.js）在浏览器中提取PDF文本：
- 前端处理PDF
- 只发送文本到后端
- 后端不需要PDF库
