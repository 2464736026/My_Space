## 后端实现

### 1. Vercel原生Python Handler

**为什么不使用FastAPI？**

最初尝试使用FastAPI + Mangum适配器，但遇到依赖冲突问题。最终采用Vercel原生Python Handler（BaseHTTPRequestHandler），优势：
- 零依赖冲突
- 完全兼容Vercel环境
- 简单高效
- 易于调试

**核心实现：**

```python
from http.server import BaseHTTPRequestHandler
import json

class handler(BaseHTTPRequestHandler):
    def do_GET(self):
        """处理GET请求"""
        if self.path == '/api/health':
            self._send_json_response(200, {
                "status": "healthy"
            })
    
    def do_POST(self):
        """处理POST请求"""
        if self.path == '/api/upload-resume':
            self._handle_upload_resume()
        elif self.path == '/api/match-job':
            self._handle_match_job()
    
    def _send_json_response(self, status_code, data):
        """发送JSON响应"""
        self.send_response(status_code)
        self.send_header('Content-type', 'application/json')
        self.send_header('Access-Control-Allow-Origin', '*')
        self.end_headers()
        self.wfile.write(json.dumps(data, ensure_ascii=False).encode('utf-8'))
```

```python
def _handle_upload_resume(self):
```

### 3. PDF解析服务

**pdf_parser.py实现：**

```python
import pdfplumber
import PyPDF2
import io

class PDFParser:
    def extract_text(self, pdf_content: bytes) -> str:
        """从PDF内容中提取文本"""
        try:
            # 首先尝试pdfplumber
            text = self._extract_with_pdfplumber(pdf_content)
            if text.strip():
                return self._clean_text(text)
            
            # 如果失败，尝试PyPDF2
            text = self._extract_with_pypdf2(pdf_content)
            return self._clean_text(text)
        except Exception as e:
            raise Exception(f"PDF解析失败: {str(e)}")
    
    def _extract_with_pdfplumber(self, pdf_content: bytes) -> str:
        """使用pdfplumber提取"""
        text_parts = []
        with pdfplumber.open(io.BytesIO(pdf_content)) as pdf:
            for page in pdf.pages:
                page_text = page.extract_text()
                if page_text:
                    text_parts.append(page_text)
        return "\n".join(text_parts)
    
    def _clean_text(self, text: str) -> str:
        """清洗文本"""
        # 移除多余空格
        text = re.sub(r'[ \t]+', ' ', text)
        # 移除特殊字符
        text = re.sub(r'[^\u4e00-\u9fff\w\s\-@.()（）：:；;，,。.！!？?\n]', '', text)
        return text.strip()
```

```python
class CacheManager:
```

## AI服务实现

### 1. 为什么不使用OpenAI Python库？

**遇到的问题：**
```
TypeError: Client.__init__() got an unexpected keyword argument 'proxies'
```

OpenAI Python库与Vercel环境中的httpx库存在深层次的兼容性问题，无论使用哪个版本都会出错。

**解决方案：**

**彻底移除OpenAI库，使用Python标准库urllib.request直接调用API！**

### 2. 核心实现

**ai_service.py：**

```python
import urllib.request
import urllib.error
import json

class AIService:
    def __init__(self):
        """初始化AI服务"""
        self.api_key = os.getenv("OPENAI_API_KEY")
        self.model = os.getenv("OPENAI_MODEL", "gpt-3.5-turbo")
        self.base_url = os.getenv("OPENAI_BASE_URL", "https://api.openai-proxy.org/v1")
    
    def _call_openai_api(self, messages: list) -> str:
        """直接调用OpenAI API"""
        url = f"{self.base_url}/chat/completions"
        
        headers = {
            "Content-Type": "application/json",
            "Authorization": f"Bearer {self.api_key}"
        }
        
        data = {
            "model": self.model,
            "messages": messages,
            "temperature": 0.1,
            "max_tokens": 2000
        }
        
        # 创建请求
        req = urllib.request.Request(
            url,
            data=json.dumps(data).encode('utf-8'),
            headers=headers,
            method='POST'
        )
        
        # 发送请求
        with urllib.request.urlopen(req, timeout=30) as response:
            result = json.loads(response.read().decode('utf-8'))
            return result['choices'][0]['message']['content']
```

### 5. JSON解析与容错

```python
def _parse_json_response(self, text: str) -> dict:
    """解析AI返回的JSON响应"""
    try:
        # 直接尝试解析
        return json.loads(text)
    except json.JSONDecodeError:
        # 提取JSON部分
        start = text.find('{')
        end = text.rfind('}')
        
        if start != -1 and end != -1:
            json_text = text[start:end+1]
            try:
                return json.loads(json_text)
            except json.JSONDecodeError:
                pass
        
        # 使用正则提取
        json_match = re.search(r'\{.*\}', text, re.DOTALL)
        if json_match:
            try:
                return json.loads(json_match.group())
            except json.JSONDecodeError:
                pass
        
        raise ValueError("无法解析AI返回的JSON格式")
```

## 遇到的问题与解决方案

### 问题1：405 Method Not Allowed

**现象：**
```
POST /api/upload-resume HTTP/1.1 405
```

**原因：**
- 后端项目缺少 `vercel.json` 配置文件
- Vercel不知道如何路由 `/api/*` 请求

**解决方案：**
创建 `vercel.json` 配置路由规则：
```json
{
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "api/index.py"
    }
  ]
}
```

**学到的经验：**
- Vercel Serverless Functions需要明确的路由配置
- `vercel.json` 必须在项目根目录（或Root Directory指定的目录）


**学到的经验：**

### 问题3：OpenAI库兼容性问题

**现象：**
```
TypeError: Client.__init__() got an unexpected keyword argument 'proxies'
```

**原因：**
- OpenAI Python库（无论哪个版本）与Vercel环境中的httpx库不兼容
- OpenAI库内部尝试传递 `proxies` 参数给httpx，但版本不匹配

**尝试的方案：**
1. ❌ 升级 `openai==1.3.7` 到 `1.54.0` → 仍然失败
2. ❌ 降级httpx版本 → 无法控制Vercel环境
3. ❌ 使用不同的OpenAI库版本 → 都有问题

**最终解决方案：**
**彻底移除OpenAI库，使用Python标准库urllib.request直接调用API！**

```python
import urllib.request

def _call_openai_api(self, messages: list) -> str:
    url = f"{self.base_url}/chat/completions"
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {self.api_key}"
    }
    data = {
        "model": self.model,
        "messages": messages,
        "temperature": 0.1,
        "max_tokens": 2000
    }
    
    req = urllib.request.Request(
        url,
        data=json.dumps(data).encode('utf-8'),
        headers=headers,
        method='POST'
    )
    
    with urllib.request.urlopen(req, timeout=30) as response:
        result = json.loads(response.read().decode('utf-8'))
        return result['choices'][0]['message']['content']
```

**优势：**
- ✅ 零依赖冲突（使用Python标准库）
- ✅ 完全兼容所有环境
- ✅ 更轻量（减少依赖数量）
- ✅ 更稳定（不受第三方库版本影响）

**学到的经验：**
- 有时候最简单的方案是最好的
- 不要过度依赖第三方库
- Python标准库功能强大且稳定
- 直接调用HTTP API比使用SDK更可控

### 问题4：Multipart/Form-Data解析

**现象：**
使用Vercel原生Handler，没有现成的multipart解析库。

**解决方案：**
手动实现multipart解析：

```python
def _handle_upload_resume(self):
    # 获取boundary
    content_type = self.headers.get('Content-Type', '')
    boundary = content_type.split('boundary=')[1].encode()
    
    # 读取请求体
    content_length = int(self.headers.get('Content-Length', 0))
    body = self.rfile.read(content_length)
    
    # 分割数据
    parts = body.split(b'--' + boundary)
    
    # 提取文件
    for part in parts:
        if b'Content-Disposition' in part and b'filename=' in part:
            # 提取文件名
            lines = part.split(b'\r\n')
            for line in lines:
                if b'filename=' in line:
                    filename = line.decode().split('filename="')[1].split('"')[0]
                    break
            
            # 提取文件内容
            content_start = part.find(b'\r\n\r\n') + 4
            content_end = len(part) - 2
            pdf_content = part[content_start:content_end]
            break
```

**学到的经验：**
- 理解HTTP协议的底层细节很重要
- multipart/form-data格式并不复杂
- 手动实现可以更好地控制和调试
