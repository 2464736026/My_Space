## 🔍 错误排查

### 错误类型1：500 Internal Server Error

**可能原因：**
- 环境变量未正确配置
- 代码运行时错误
- 依赖包安装失败

**解决方法：**
1. 在Vercel Dashboard中点击最新的部署
2. 查看 "Function Logs" 标签
3. 找到包含 "ERROR" 或 "Traceback" 的日志
4. 将完整错误信息发送给我

### 错误类型2：404 Not Found

**可能原因：**
- 路由配置问题
- 函数未正确部署

**解决方法：**
1. 确认 `vercel.json` 在项目根目录
2. 在Vercel Dashboard查看 "Functions" 标签
3. 确认有 `api/index.py` 函数

### 错误类型3：环境变量问题

**症状：** `/api/health` 返回 `"openai_configured": false`

**解决方法：**
1. 进入 Vercel Dashboard → 项目 → Settings → Environment Variables
2. 确认以下变量存在：
   - `OPENAI_API_KEY` = `sk-R4Z3s07Fvg0nuvLlLtmMNe7EGRhv8DLhmlicxy9x6tEPRd7m`
   - `OPENAI_MODEL` = `gpt-3.5-turbo`
   - `OPENAI_BASE_URL` = `https://api.openai-proxy.org/v1`
3. 确认这些变量应用到了 "Production" 环境
4. 点击 "Redeploy" 按钮重新部署

## 📊 查看详细日志

### 在Vercel Dashboard中：

1. 进入项目页面
2. 点击 "Deployments" 标签
3. 点击最新的部署
4. 查看以下标签：
   - **Build Logs**: 构建过程的日志
   - **Function Logs**: 运行时日志（最重要！）
   - **Runtime Logs**: 实时运行日志

### 查找关键信息：

在日志中搜索：
- `ERROR`
- `Traceback`
- `Failed`
- `Exception`
- `Initializing AIService` (我们添加的日志)
- `OPENAI_API_KEY exists` (我们添加的日志)
