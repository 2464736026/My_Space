# 依赖打包说明

## 问题原因

从日志看到错误：
```
ModuleNotFound: No module named 'fastapi'
```

这是因为上传的代码包**没有包含依赖库**。

阿里云函数计算不会自动安装 `requirements.txt` 中的依赖，需要将依赖一起打包上传。

---

## ✅ 解决方案

我已经更新了 `deploy_aliyun.bat`，现在会：
1. 自动安装依赖到 `python` 目录
2. 将依赖一起打包到 ZIP 文件中

---

## 📝 重新打包步骤

### 第一步：运行新的打包脚本

```cmd
cd E:\newP\backend
deploy_aliyun.bat
```

**注意**：
- 第一次运行会下载并安装所有依赖
- 可能需要 5-10 分钟
- 最终的 ZIP 文件会比较大（50-100MB）

### 第二步：上传到阿里云

1. 访问：https://fcnext.console.aliyun.com/cn-hangzhou/functions
2. 选择函数 `api`
3. 点击"代码"标签
4. 点击"上传代码" → "上传 ZIP 包"
5. 选择新的 `resume-analyzer.zip`
6. **点击"部署"**
7. 等待部署完成（可能需要 1-2 分钟，因为文件较大）

### 第三步：测试

访问：
```
https://api-lanxianlei.cn-hangzhou.fcapp.run/
```

应该看到：
```json
{
  "message": "AI Resume Analyzer API",
  "version": "1.0.0"
```

## ⚠️ 注意事项

### 1. 文件大小限制

阿里云函数计算的限制：
- 直接上传：最大 50MB
- 通过 OSS：最大 500MB

如果打包后超过 50MB，需要：
1. 使用阿里云 OSS 上传
2. 或者精简依赖

### 2. 依赖安装要求

确保本地环境：
- ✅ 已安装 Python 3.9 或 3.10
- ✅ 已安装 pip
- ✅ 网络连接正常（需要下载依赖）

### 3. 如果打包失败

可能的原因：
- pip 未安装：`python -m ensurepip`
- 网络问题：使用国内镜像源
- 权限问题：以管理员身份运行

---

## 💡 使用国内镜像加速

如果下载依赖很慢，可以使用清华镜像：

```cmd
pip install -r requirements.txt -t python -i https://pypi.tuna.tsinghua.edu.cn/simple
```
