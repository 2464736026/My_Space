# 前端部署指南

## 📋 项目结构

```
My_Space/
├── api/                    # 后端API（已部署到Vercel）
│   ├── index.py           # 后端入口
│   ├── services/          # AI和PDF服务
│   └── requirements.txt   # Python依赖
│
├── frontend/              # 前端项目（需要部署）
│   ├── src/              # React源代码
│   │   └── App.jsx       # 主应用
│   ├── package.json      # Node依赖
│   ├── vite.config.js    # Vite配置
│   └── vercel.json       # Vercel配置
│
└── vercel.json           # 后端Vercel配置
```

## 🎯 部署目标

- **后端API**: https://my-space-beryl.vercel.app （已完成✅）
- **前端界面**: 需要部署到新的Vercel项目

## 🚀 部署方法

### 重要说明

由于后端和前端在同一个仓库 `My_Space`，你需要：
- ✅ 后端项目：已存在，使用根目录（`api/`）
- 🆕 前端项目：需要新建，使用 `frontend/` 目录

**不需要重新导入仓库**，只需要创建第二个Vercel项目，指向同一个仓库的不同目录。

---

### 方法1：通过Vercel CLI（推荐）

这是最简单可靠的方法，会自动创建新项目。

#### 步骤1：进入frontend目录

```powershell
cd E:\newP\frontend
```

#### 步骤2：确认环境

```powershell
# 确认package.json存在
dir package.json

# 确认Vercel CLI已安装
vercel --version
```

如果未安装Vercel CLI：
```powershell
npm install -g vercel
```

#### 步骤3：登录Vercel

```powershell
vercel login
```

#### 步骤4：部署

```powershell
vercel --prod
```

#### 步骤5：按提示回答

| 提示 | 回答 |
|------|------|
| Set up and deploy? | `Y` |
| Which scope? | 直接回车（使用默认账号）|
| Link to existing project? | `N` |
| What's your project's name? | `my-space-frontend` |
| In which directory is your code located? | 直接回车（`./`）|
| Want to override the settings? | `N` |

#### 步骤6：等待完成

约2-3分钟后，会显示：
```
✅ Production: https://my-space-frontend-xxx.vercel.app
```

---

### 方法2：通过Vercel Dashboard

如果CLI有问题，可以使用网页界面创建第二个项目。

#### 步骤1：访问Vercel

打开：https://vercel.com/dashboard

#### 步骤2：创建新项目（不是导入新仓库）

1. 点击 "Add New..." → "Project"
2. 在项目列表中找到 **`My_Space`** 仓库
3. 点击 "Import" 按钮

**注意**：
- 你会看到 `My_Space` 仓库已经有一个项目（后端）
- 这是正常的，Vercel允许同一个仓库创建多个项目
- 每个项目可以配置不同的Root Directory

#### 步骤3：配置项目

**重要配置：**

| 配置项 | 值 |
|--------|-----|
| Project Name | `my-space-frontend` |
| Framework Preset | **Vite** |
| Root Directory | **frontend** （通过文件选择器选择）|
| Build Command | `npm run build` |
| Output Directory | `dist` |

**⚠️ Root Directory配置方法：**
1. 点击 "Edit" 按钮
2. 在文件选择器中点击 `frontend` 文件夹
3. 点击 "Continue"
4. 确认显示为 `frontend`

#### 步骤4：部署

点击 "Deploy" 按钮，等待完成。

## 🔧 故障排查

### 问题1：找不到package.json

**错误信息：**
```
npm error enoent Could not read package.json
```

**原因：** 不在正确的目录或Root Directory配置错误

**解决方案：**
- CLI方法：确保在 `E:\newP\frontend` 目录执行
- Dashboard方法：重新配置Root Directory

### 问题2：构建失败

**错误信息：**
```
Error: Build failed
```

**解决方案：**
```powershell
# 本地测试构建
cd E:\newP\frontend
npm install
npm run build

# 如果本地成功，再部署
vercel --prod
```

### 问题3：依赖安装失败

**解决方案：**
```powershell
# 清理并重新安装
cd E:\newP\frontend
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install
vercel --prod
```
