# 后端405问题深度诊断

## 🔍 问题现象

1. ✅ 前端可以访问
2. ✅ 后端 `/api/health` 和 `/api/test` 可以访问（GET请求）
3. ❌ 后端 `/api/upload-resume` 返回405错误（POST请求）
4. ❌ 后端没有任何日志输出

## 🎯 关键发现

**405错误 + 无日志 = 请求根本没有到达Python代码**

这说明问题在Vercel的路由层面，而不是Python代码层面。

## 🔧 可能的原因

### 原因1：Vercel项目配置了Root Directory

如果后端项目在Vercel Dashboard中配置了Root Directory（比如设置为 `api/`），那么：

```
实际文件路径: /api/index.py
Vercel看到的路径: /index.py (因为Root Directory是api/)
vercel.json中的路由: /api/(.*)
结果: 路由不匹配！
```

**检查方法：**
1. 访问 Vercel Dashboard
2. 找到后端项目（my-space 或 my-space-beryl）
3. Settings → General → Root Directory
4. 查看是否设置了值

**解决方案：**
- 如果Root Directory是 `api/`，需要修改 `vercel.json` 的路由
- 或者将Root Directory设置为空（使用根目录）

### 原因2：vercel.json位置错误

如果后端项目的Root Directory是 `api/`，那么 `vercel.json` 应该在 `api/` 目录下，而不是项目根目录。

**当前结构：**
```
My_Space/
├── vercel.json          ← 在这里
├── api/
│   └── index.py
```

**如果Root Directory是api/，应该是：**
```
My_Space/
├── api/
│   ├── vercel.json      ← 应该在这里
│   └── index.py
```

### 原因3：路由优先级问题

Vercel的路由是按顺序匹配的，可能被其他规则拦截了。

## 🚀 解决方案

### 方案A：清空Root Directory（推荐）

1. 访问 Vercel Dashboard
2. 进入后端项目设置
3. Settings → General → Root Directory
4. 清空或设置为 `.`（根目录）
5. 保存并重新部署

这样 `vercel.json` 在根目录就能正常工作。

### 方案B：移动vercel.json到api目录

如果Root Directory必须是 `api/`，那么：

1. 将 `vercel.json` 移动到 `api/` 目录
2. 修改路由配置：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "index.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "index.py"
    }
  ]
}
```

注意：路径变成了相对于 `api/` 目录的路径。

### 方案C：使用Serverless Functions目录结构

Vercel有一个特殊的 `api/` 目录约定：

```
项目根目录/
├── api/
│   ├── upload-resume.py    ← 自动映射到 /api/upload-resume
│   ├── match-job.py        ← 自动映射到 /api/match-job
│   └── health.py           ← 自动映射到 /api/health
```

这种方式不需要 `vercel.json`，Vercel会自动处理路由。
