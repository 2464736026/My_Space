# 405错误修复说明

## 🔍 问题诊断

### 错误信息
```
Uncaught (in promise) Error: Request failed with status code 405
Method: POST
URL: https://my-space-beryl.vercel.app/api/upload-resume
```

### 问题原因

**405 Method Not Allowed** 错误表示服务器不允许使用POST方法访问该路径。

根本原因：
1. ❌ 后端项目根目录缺少 `vercel.json` 配置文件
2. ❌ Vercel不知道如何路由 `/api/*` 请求到 `api/index.py`
3. ❌ 导致所有API请求返回405错误

## ✅ 已修复的问题

### 1. 创建 vercel.json 配置文件

在项目根目录创建了 `vercel.json`：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "api/index.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "api/index.py"
    },
    {
      "src": "/(.*)",
      "dest": "api/index.py"
    }
  ]
}
```

**配置说明：**
- `builds`: 告诉Vercel使用Python运行时构建 `api/index.py`
- `routes`: 定义路由规则
  - `/api/*` → 转发到 `api/index.py`
  - `/*` → 所有其他请求也转发到 `api/index.py`（处理根路径）

## 📊 技术细节

### Vercel路由工作原理

```
用户请求: POST /api/upload-resume
    ↓
Vercel路由匹配: /api/(.*)
    ↓
转发到: api/index.py (Python Handler)
    ↓
handler.do_POST() 处理请求
    ↓
返回JSON响应
```

### 为什么之前会出现405错误？

没有 `vercel.json` 时：
```
用户请求: POST /api/upload-resume
    ↓
Vercel: 找不到路由配置
    ↓
返回: 405 Method Not Allowed
```

有 `vercel.json` 后：
```
用户请求: POST /api/upload-resume
    ↓
Vercel: 匹配到路由规则 /api/(.*)
    ↓
转发到: api/index.py
    ↓
成功处理: 200 OK
```
