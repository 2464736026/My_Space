# 🚀 阿里云部署步骤（简化版）

## 问题分析

你遇到的错误 `"required HTTP header Date was not specified"` 是因为阿里云函数计算的 HTTP 触发器对 WSGI 响应格式有特殊要求。

## ✅ 解决方案

我已经简化了 `backend/index.py`，使用最简单的 WSGI 适配器。

---

## 📝 部署步骤

### 1. 打包代码

双击运行：
```
backend\deploy_aliyun.bat
```

这会生成 `backend\resume-analyzer.zip`

### 2. 上传到阿里云

1. 访问：https://fcnext.console.aliyun.com/cn-hangzhou/functions
2. 选择你的函数（例如：`api`）
3. 点击"代码"标签
4. 点击"上传代码" → "上传 ZIP 包"
5. 选择 `backend\resume-analyzer.zip`
6. **重要**：点击"部署"按钮
7. 等待部署完成

### 3. 配置环境变量

在"配置"标签中，点击"编辑"，添加：

```
OPENAI_API_KEY = sk-R4Z3s07Fvg0nuvLlLtmMNe7EGRhv8DLhmlicxy9x6tEPRd7m
OPENAI_MODEL = gpt-3.5-turbo
OPENAI_BASE_URL = https://api.openai-proxy.org/v1
```

点击"保存"，然后再次点击"部署"

### 4. 确认基本配置

在"配置"标签中确认：

- **函数入口**：`index.handler`
- **运行环境**：`Python 3.10`
- **内存规格**：`1024 MB`
- **超时时间**：`60 秒`

如果不对，点击"编辑"修改，保存后重新部署。

### 5. 测试

访问你的FC地址（在"触发器"标签中复制），应该看到：

```json
{
  "message": "AI Resume Analyzer API",
  "version": "1.0.0"
}
```

## 🔍 如果还是失败

### 查看日志

1. 在函数详情页点击"日志查询"
2. 选择最近15分钟
3. 点击"查询"
4. 查看错误信息并告诉我

### 常见问题

1. **上传后没有点击"部署"** → 必须点击部署才能生效
2. **环境变量未配置** → 必须配置3个环境变量
3. **函数入口错误** → 必须是 `index.handler`
4. **内存不足** → 至少需要 512MB，推荐 1024MB
